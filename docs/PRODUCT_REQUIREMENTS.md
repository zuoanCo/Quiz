# 刷题助手产品文档 v0.1

## 1. 项目背景

刷题助手是一款面向本地题库管理和高频练习的桌面端应用。项目目标是交付给刘总一套可验收、可继续扩展的 Electron 程序：既能离线维护本地题库，也能为后续远程题库接入留出稳定接口。

产品参考 letscode、驾考宝典等练习类应用，但不直接复制其复杂信息密度。首版重点是题库维护方便、练习路径短、提交后答案和解析清晰，整体界面采用 macOS 风格，保持干净、易用、有高级感，避免杂乱。

## 2. 目标用户

### 2.1 题库维护者

- 教师、培训机构运营、企业内训管理员。
- 关注题目录入效率、分类管理、批量导入、数据备份和远程题库对接。

### 2.2 刷题用户

- 学生、考试备考用户、企业培训学员。
- 关注快速进入练习、答案解析清晰、练习记录可追踪。

## 3. 产品目标

1. 支持单选、多选、判断、填空、简答、论述/分析六类题目。
2. 支持科目和章节管理，章节可选。
3. 支持手动上传题目和题目文本自动解析，并为批量导入留出入口。
4. 支持随机练习和章节练习。
5. 提交后展示正确答案和解析，首版可以不自动判定正确与否。
6. 默认本地题库可用，断网不影响核心刷题。
7. 代码结构预留远程题库能力，后续可对接服务端 API。
8. 桌面端体验接近 macOS 原生应用，界面克制、清爽、信息层次明确。

## 4. 功能范围

### 4.1 MVP 必做

#### 科目管理

- 新增科目。
- 编辑科目名称。
- 删除科目。
- 查看科目下题目数量和章节数量。

#### 章节管理

- 在指定科目下新增章节。
- 编辑章节名称。
- 删除章节。
- 章节为可选字段，题目可只属于科目。

#### 题目管理

- 支持题型：单选、多选、判断、填空、简答、论述/分析。
- 题目字段：
  - 题型。
  - 所属科目。
  - 所属章节。
  - 题干。
  - 选项。
  - 正确答案。
  - 解析。
  - 标签或难度预留。
- 支持题目列表查看、筛选和搜索。
- 支持新增、编辑、删除题目。

#### 题目自动解析

- 支持粘贴原始题目文本，系统自动识别题干、选项、正确答案和解析。
- 支持单题解析，也为后续多题批量解析预留结构。
- 解析后进入草稿预览，用户确认或修正后再保存入库。
- 支持常见格式：
  - `【题干】...`
  - `A. ... / B. ... / C. ... / D. ...`
  - `答案：A`
  - `正确答案：A、C`
  - `解析：...`
- 对无法识别的字段给出提示，不强行入库。
- 非选择题可自动识别参考答案、答案要点和解析；若无法判断题型，默认进入人工确认。

#### 随机练习

- 可选择全部题库或某个科目。
- 从匹配题目中随机抽取。
- 用户作答后点击提交。
- 提交后展示正确答案和解析。
- 支持下一题。

#### 章节练习

- 选择科目和单个章节题单。
- 展示章节题量、题型分布和当前进度。
- 按章节题单顺序推进，支持上一题、下一题和本章重练。
- 作答、提交和解析复盘体验与题型保持一致。

#### 本地题库

- 应用本机持久化保存数据。
- 首版数据不依赖远程服务。
- 支持未来导出备份。

#### 远程题库预留

- 提供远程题库设置页。
- 支持配置 API 地址、访问令牌、连接状态。
- 代码层提供本地仓库和远程仓库接口。
- MVP 可先做模拟连接和结构预留，后续接真实服务端。

### 4.2 V1 增强

- CSV、Excel、JSON 批量导入。
- 多题连续文本自动拆题和解析。
- 题库导出和备份恢复。
- 练习记录。
- 收藏题、错题本。
- 简答题自评：掌握、模糊、未掌握。
- 远程题库拉取、上传、同步冲突提示。
- 多端同步和账号体系。

## 5. 信息架构

应用采用左侧侧边栏 + 顶部标题区 + 主工作区结构。

### 5.1 侧边栏

- 今日概览。
- 随机练习。
- 章节练习。
- 题库管理。
- 科目与章节。
- 导入导出。
- 远程题库。

### 5.2 主工作区

不同模块只展示当前任务需要的信息，避免一屏塞入过多设置项。

- 今日概览：题库统计、最近练习、快捷入口。
- 练习页：题干、选项或简答输入、提交区、答案解析区。
- 题库管理：题目列表、筛选器、编辑面板。
- 科目章节：科目列表、章节列表、数量统计。
- 远程题库：连接配置、同步状态、操作入口。

## 6. 核心流程

### 6.1 建库流程

1. 创建科目。
2. 可选创建章节。
3. 新增题目。
4. 填写题干、选项、答案、解析。
5. 保存题目。
6. 在题库管理中查看和筛选。

### 6.2 随机练习流程

1. 进入随机练习。
2. 选择练习范围。
3. 点击开始。
4. 系统抽取一道题。
5. 用户选择或填写答案。
6. 点击提交。
7. 展示正确答案和解析。
8. 点击下一题继续。

### 6.3 章节练习流程

1. 进入章节练习。
2. 选择科目。
3. 选择一个章节题单。
4. 系统展示章节题量、题型分布和当前进度。
5. 用户按章节题单顺序作答、复盘，并可在章节末尾重练本章。

### 6.4 远程题库流程

1. 进入远程题库设置。
2. 填写服务地址和访问令牌。
3. 测试连接。
4. 拉取远程题库到本地缓存。
5. 本地优先练习，后续支持同步。

## 7. 题型设计

### 7.1 单选题

- 选项 A、B、C、D 等。
- 用户只能选择一个选项。
- 答案存储为单个选项 key。

### 7.2 多选题

- 选项 A、B、C、D 等。
- 用户可选择多个选项。
- 答案存储为选项 key 数组。

### 7.3 简答题

- 用户输入文本答案。
- 提交后展示参考答案和解析。
- MVP 不做智能判分。

## 9. 视觉与体验原则

1. macOS 风格：左侧轻量侧边栏、圆角窗口感、柔和分割线、克制阴影。
2. 清爽优先：一屏只突出一个主要任务。
3. 题目优先：练习页以题干和选项为核心，解析在提交后再展开。
4. 上传方便：题目录入页采用分区表单，减少长表单压迫感。
5. 信息不过载：统计数据只保留必要数字，避免营销式大面积卡片。
6. 高级感来自细节：字体、间距、边框、动效、状态反馈保持统一。

## 10. 技术方案

### 10.1 客户端

- Electron：桌面端容器。
- React：前端界面。
- TypeScript：类型约束。
- Vite：开发构建。
- CSS Modules 或全局设计变量：实现 macOS 风格视觉系统。

### 10.2 数据层

MVP 可采用浏览器本地存储抽象实现快速交付，代码结构保留 Repository 接口。正式版本建议切换 SQLite。

Repository 设计：

- QuestionRepository：题目增删改查。
- SubjectRepository：科目增删改查。
- PracticeRepository：练习记录。
- RemoteBankRepository：远程题库接口。

### 10.3 远程接口预留

推荐 API：

- GET /subjects
- GET /subjects/:id/chapters
- GET /questions
- POST /questions
- PUT /questions/:id
- DELETE /questions/:id
- POST /sync/pull
- POST /sync/push

## 11. 验收标准

1. 可以启动 Electron 程序。
2. 可以新增、编辑、删除科目。
3. 可以为科目新增、编辑、删除章节。
4. 可以新增单选、多选、判断、填空、简答、论述/分析题。
5. 题目可以保存到本地并在重启后保留。
6. 随机练习可以抽取题目。
7. 章节练习可以按单个章节题单推进，并展示章节进度。
8. 提交后展示正确答案和解析。
9. 远程题库设置页存在，并有连接配置和状态反馈。
10. UI 风格符合 macOS 桌面应用观感，页面不杂乱。
